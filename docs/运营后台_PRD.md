# 广交会项目 - 运营后台 PRD（评审版）

> 版本：V0.1  
> 日期：2026-03-04  
> 范围：运营后台（admin）  
> 模块：大盘架构集（配送管理、配送统计、小程序管理看板预览）

## 1. 项目目标

构建统一的运营后台，支撑广交会餐饮履约场景的“配置可控 + 过程可见 + 结果可追溯”。

核心目标：
1. 配置层：快速完成点位、商家、配送员责任范围配置。
2. 监控层：实时掌握配送状态、拥堵风险和异常单。
3. 复盘层：按日期与维度回看历史统计，支撑运营调优。

## 2. 用户角色

1. 超级管理员：全域配置、权限管理、审计查看。
2. 运营配置人员：维护配置与日常巡检。
3. 区长/配送主管：查看本区履约与人效数据，处理异常。
4. 只读角色：仅查看统计与责任分布。

## 3. 模块边界

### 3.1 模块A：配送管理

能力范围：
1. 点位-商家配置（一个点位对多个商家）。
2. 配送员账号增删（手机号唯一）。
3. 主负责范围批量配置（展区必选，楼层/展厅可选）。
4. 主责任查询（按展区、楼层、展厅筛选）。

### 3.2 模块B：PC端配送统计

能力范围：
1. 实时配送大盘与多维切片（当前单量、完成率、超时风险）。
2. 历史统计复杂查询（日期、展区、楼层、展厅、配送员、餐段、订单状态）。
3. 送达阶段全维度漏斗分布（待接单/赴集散/在集散/送展位/已送达）。
4. 异常单与超时单重灾区明细检索与导出。

### 3.3 模块C：配送小程序_管理看板

能力范围：
1. 针对指挥长、运营负责人的在场型手机端核心督战主页。
2. 根据时间刻度瀑布流划分进行中、严重烂尾、未开始与已完结批次。
3. 四段流速的动态刷新，以及下沉至具体“送集散地超时”、“待接单延误”等红黑榜单报警。
4. 在线人力排查及“当前空闲运力”实时抽提明细展示。
5. 集成原生的“一键实物释放”异常包裹逆流程特权操作台。

## 4. 核心业务规则

1. 配送阶段口径统一：待接单、赴集散、在集散、送展位、已送达。
2. 配置继承口径统一：只选展区=全楼层全展厅；选楼层未选展厅=该楼层全展厅。
3. 查询口径统一：展厅查询需同时可见 HALL/FLOOR/AREA 三层命中责任人。
4. 统计口径统一：实时数据与历史数据字段同源，同一筛选器结构。

## 5. 验收标准（平台级）

1. 进入 `admin` 后能够打通查阅从 PC管理配置 到 移动端看板预览 的全部路由网。
2. 配送管理模块可完成配置闭环并可按区查询到责任人。
3. 配送统计及看板预估均与数据库同频同口径渲染。
4. 各个页面均支持异常态、空态打底并符合规范。
5. 文档体系完整：顶层 `运营后台_PRD.md` 将负责统筹，三大 `modules` 自辖底层 PRD/Spec/IA 构成规范工程。

## 6. 非目标

1. 不包含自动派单引擎逻辑实现。
2. 不包含配送小程序侧（C端骑手界面）的其他普通流转接单界面。
3. 不包含财务结算与商户分账等与履约无关的后置链路。

## 7. 里程碑建议

1. M1：文档冻结（PRD/Spec/IA）。
2. M2：高保真原型评审通过。
3. M3：开发联调与UAT。
4. M4：灰度上线与复盘。
