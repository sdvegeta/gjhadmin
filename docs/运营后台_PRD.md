# 广交会项目 - 运营后台 PRD（评审版）

> 版本：V0.1  
> 日期：2026-02-28  
> 范围：运营后台（admin）  
> 模块：配送管理、配送统计

## 1. 项目目标

构建统一的运营后台，支撑广交会餐饮履约场景的“配置可控 + 过程可见 + 结果可追溯”。

核心目标：
1. 配置层：快速完成点位、商家、配送员责任范围配置。
2. 监控层：实时掌握配送状态、拥堵风险和异常单。
3. 复盘层：按日期与维度回看历史统计，支撑运营调优。

## 2. 用户角色

1. 超级管理员：全域配置、权限管理、审计查看。
2. 运营配置人员：维护配置与日常巡检。
3. 区长/配送主管：查看本区履约与人效数据，处理异常。
4. 只读角色：仅查看统计与责任分布。

## 3. 模块边界

### 3.1 模块A：配送管理

能力范围：
1. 点位-商家配置（一个点位对多个商家）。
2. 配送员账号增删（手机号唯一）。
3. 主负责范围批量配置（展区必选，楼层/展厅可选）。
4. 主责任查询（按展区、楼层、展厅筛选）。

### 3.2 模块B：配送统计

能力范围：
1. 实时配送总览（当前单量、完成率、超时风险）。
2. 历史统计查询（日期、展区、楼层、展厅、配送员、餐段、订单状态）。
3. 配送阶段分布与趋势（待接单/赴集散/在集散/送展位/已送达）。
4. 异常单与超时单明细（支持导出）。

## 4. 核心业务规则

1. 配送阶段口径统一：待接单、赴集散、在集散、送展位、已送达。
2. 配置继承口径统一：只选展区=全楼层全展厅；选楼层未选展厅=该楼层全展厅。
3. 查询口径统一：展厅查询需同时可见 HALL/FLOOR/AREA 三层命中责任人。
4. 统计口径统一：实时数据与历史数据字段同源，同一筛选器结构。

## 5. 验收标准（平台级）

1. 进入 `admin` 后可在同一后台访问两大模块。
2. 配送管理模块可完成配置闭环并可查询责任人。
3. 配送统计模块可按指定维度筛选并展示实时与历史结果。
4. 页面支持加载、空态、错误、权限不足、二次确认等关键状态。
5. 文档体系完整：运营后台整体 + 两个模块均具备 PRD/Spec/IA。

## 6. 非目标

1. 不包含自动派单引擎。
2. 不包含配送小程序端改造实现。
3. 不包含财务结算与分账流程实现。
