# 广交会项目 - 后台配送统计页 PRD（评审版）

> 版本：V0.2  
> 日期：2026-03-01  
> 模块：运营后台 / 配送统计

## 1. 目标

实现“餐段内可即时调度、餐段外可复盘优化”的统计能力，帮助配送主管快速判断当前进度与履约风险。

## 2. 用户与场景

1. 配送主管/区长：餐段内实时识别堵点并调度。
2. 运营：按展区/展厅/商家定位履约瓶颈。
3. 管理层：按日/餐段评估履约质量和改进效果。

## 3. 核心功能

### 3.1 统计筛选器

1. 日期
2. 展区
3. 楼层
4. 展厅
5. 配送员
6. 餐段
7. 订单状态

### 3.2 实时看板（餐段内）

1. 阶段单量与占比（待接单/赴集散/在集散/送展位/已送达）。
2. 超时风险（超时未送达单量及占比、超时送达单量及占比）。
3. 阶段超时（四段停留超时单量及占比：10/10/10/25）。
4. 波次进度（当前波次总单、已送达、完成率、剩余单量）。
5. Top堵点三榜：
   - Top3 商家待接堵点
   - Top3 展厅在集散堵点
   - Top3 展厅送展位堵点

### 3.3 餐段外复盘（昨日/前日）

1. 结果质量：波次准时达成率、超时未送达率、超时送达率。
2. 过程效率：各阶段停留时长（均值/P90）。
3. 结构复盘：商家待接超时率 TopN，展厅在集散/送展位超时率 TopN。
4. 人员效率：配送员完成单量、配送员准时送达率。
5. 波次治理：波次关账一次通过率。

## 4. 业务规则

1. 状态字典固定为 5 段：待接单、赴集散、在集散、送展位、已送达。
2. 阶段超时阈值固定：待接 10 分钟、赴集散 10 分钟、在集散 10 分钟、送展位 25 分钟。
3. 阶段占比默认分母为当前餐段总单量；阶段超时占比分母为该阶段当前单量。
4. Top堵点窗口默认最近 15 分钟，排序规则为“超时单量降序 + 最老超时时长降序”。
5. 本期不包含异常池与异常单处置功能，仅统计阶段超时与履约结果。

## 5. 非目标（本期不做）

1. 不包含异常池管理与异常单导出。
2. 不包含机动人员调度能力展示。
3. 不包含自动改派策略。

## 6. 验收标准

1. 筛选器字段可用且联动正确。
2. 5 段状态单量合计等于当前餐段总单量。
3. 四阶段超时统计口径与阈值一致（10/10/10/25）。
4. 波次进度字段可用于直接计算完成率与剩余单量。
5. Top堵点榜单可按规则稳定排序并支持钻取。
6. 餐段外复盘指标可按昨日/前日成功查询。
