# 广交会项目 - 后台配送管理界面 PRD（评审版）

> 版本：V0.5  
> 日期：2026-04-09  
> 适用范围：运营后台「配送管理界面」子项目（不含配送统计页、不含配送小程序筛选页）  
> 产出目标：为高保真原型与研发拆解提供一致业务基线

## 1. 业务背景

广交会场景下，配送组织采用“同区履约 + 分组负责 + 分段配送”主干逻辑。为保证峰值期可控履约，需要在运营后台提供统一的配送配置入口，解决以下问题：

1. 点位与商家供应关系需可配置，避免点位出现无可服务商家或商家跨区错配。
2. 配送员账号需快速创建与维护，满足大促期临时扩编。
3. 配送员“主负责范围”需结构化配置（展区/楼层/展厅），并支持批量配置。
4. 运营与区长需按展区-楼层-展厅快速查询当前主责任配送人员。
5. 配送团队负责人需在用户餐段截单前，按展区查看各商家的实时订单量与餐品份数，用于提前安排运力。

## 2. 产品目标与非目标

### 2.1 产品目标

1. 建立点位-商家映射能力，支撑排期时点位只显示可服务商家。
2. 建立配送员账号管理能力，手机号唯一，账号创建流程最简。
3. 建立“主负责范围”配置能力，支持多人批量配置与层级继承规则。
4. 建立主责查询能力，按展区/楼层/展厅可视化查询责任人。
5. 建立“订单实时监控”能力，按展区查看各商家在当前开放餐段下的实时订单量。

### 2.2 非目标（本期不做）

1. 不包含配送统计页建设。
2. 不包含配送小程序筛选模块改造。
3. 不包含自动派单算法与实时调度策略改造。
4. 不包含跨区支援策略自动化（仅保留规则字段与人工扩展能力）。
5. 不在本页承载完整配送统计大盘、四段漏斗与历史复盘。

## 3. 目标用户与角色

1. 运营配置人员：维护点位-商家关系、配送员账号、主负责范围。
2. 区长/配送主管：按区域查看主责人、检查配置完整性。
3. 超级管理员：全局配置与审计追踪。
4. 配送团队负责人/区长：查看订单实时监控并据此做人手安排。

## 4. 核心业务规则

### 4.1 点位-商家映射规则

1. 一个点位可绑定多个商家。
2. 同一点位禁止重复绑定同一商家。
3. 排期/下游使用点位时，仅返回该点位已绑定商家。

### 4.2 配送员账号规则

1. 新增配送员仅需姓名、手机号，默认初始角色为【配送员】。
2. 手机号为全局唯一键，重复手机号禁止保存。但如遇已停用的手机号重新入职：支持检测并原地弹窗激活（即软删除唤醒替换）。
3. 提供“停用”和“恢复”机制控制账号活性。
4. 提供“配置角色”能力。支持【配送员】【分拣签收员】【配送主管】三种枚举。
5. 角色分配防呆规则：必须保留至少1个角色；【配送主管】可与任一角色共存。
6. **全局权限例外**：【分拣签收员】对所有展厅点位和商家天生具备全部操作权限，因此在主责配置表中不可被勾选分配。

### 4.3 主负责范围配置规则

1. 配置维度为：展区-楼层-展厅。界面要求合理利用空间，保证所有楼层和展厅均为**显性平铺选择**。
2. 三层均支持多选，弹窗内必须包含一键“全选”交互，支持批量选人后一次配置。
3. **严格的全量覆盖逻辑**：配置保存时直接使用界面勾选的最新节点集完全替换旧数据，并在操作弹窗内提供显性的“覆盖原有配置”蓝色背景+红色字体的强制警示。
4. 同一配送员可配置多条主责展厅与责任商家记录。

### 4.4 主责查询规则

1. 支持按展区、楼层、展厅、商家名称快速筛选查询主责任配送员。
2. 查询结果在 `配送员主责配置` 的统一列表中展示，支持直观显示对应配送员的：**姓名、手机号、角色、主责展厅点位、主责商家**。

### 4.5 订单实时监控规则

1. 页面按展区分组展示，每个展区支持展开查看对应商家。
2. 列维度固定为：`9点截单`、`10点截单`、`10:30截单`、`11:00截单`、`即时出单`。
3. 统计对象为用户下单后形成的购物车订单，排除已取消订单。
4. 展示字段为：
   - 购物车订单数：用户在同一商家中，购物车提交并支付成功一次，记为 1 个购物车订单。
   - 餐品份数：该购物车订单内全部餐品份数之和。
5. 用户端同一时刻仅开放一个可下单餐段，因此监控页在任一时刻只应有一个“当前开放餐段”作为主增长列：
   - `08:50` 前：`9点截单`
   - `08:50` ~ `09:49`：`10点截单`
   - `09:50` ~ `10:19`：`10:30截单`
   - `10:20` ~ `10:49`：`11:00截单`
   - `10:50` 后：`即时出单`
6. 已切走餐段仅保留历史沉淀量，不再作为当前关注列；未开放餐段默认无订单。
7. 页面固定 60 秒自动刷新一次，并支持手动刷新；与上次刷新相比发生变化的数字以红色高亮。

## 5. 页面模块划分（后台配送管理界面）

### 5.1 P0-01 订单实时监控

- 页面定位：配送团队负责人在截单前查看各展区、各商家的实时订单情况，用于提前安排运力。
- 页面目标：
1. 让负责人快速识别当前开放餐段对应的高单量展区和商家。
2. 让负责人基于订单数与餐品份数，提前判断是否需要调配人手。
3. 通过轻量刷新与变化高亮，降低人工反复比对成本。

### 5.2 P0-02 配送员主责配置

- 页面定位：统一查阅并批量配置配送员负责的展厅点位与责任商家。
- 页面目标：
1. 支持多人批量配置主责范围。
2. 支持按展区、楼层、展厅、商家快速回查责任人。

### 5.3 P0-03 点位-商家配置

- 页面定位：维护点位与商家的映射关系，约束下游排期可选商家范围。
- 页面目标：
1. 保证点位不会出现无可服务商家。
2. 保证排期时仅露出已配置商家。

### 5.4 P0-04 配送员名册

- 页面定位：维护配送员账号、角色与账号状态。
- 页面目标：
1. 支持快速新增、停用、恢复配送员账号。
2. 支持角色配置与人员基础信息维护。

## 7. 页面详细说明

### 7.1 P0-01 订单实时监控

#### 7.1.1 页面结构

1. 顶部信息条：
   - 最新刷新时间
   - 自动刷新倒计时（60秒）
   - 立即刷新按钮
2. 主体内容区：
   - 按展区分组展示
   - 展区支持折叠/展开
   - 展开后展示该区各商家在不同截单时段下的横向表格
3. 表格底部：
   - 展区小计

#### 7.1.2 表格字段

1. 行维度：
   - 商家
   - 展区小计
2. 列维度：
   - `9点截单`
   - `10点截单`
   - `10:30截单`
   - `11:00截单`
   - `即时出单`
3. 单元格字段：
   - 订单数
   - 餐品份数

#### 7.1.3 视觉表达规则

1. 当前开放餐段列：轻量强调。
2. 已切走餐段列：轻量降权。
3. 未开放餐段列：保持常规样式，但数值为 0。
4. 与上次刷新相比发生变化的数字：红色高亮。
5. 不额外增加“未截单/已截单”文字说明。

#### 7.1.4 时间切换规则

1. `08:50` 前：用户端仅开放 `9点截单` 餐段。
2. `08:50` ~ `09:49`：用户端切换到 `10点截单`。
3. `09:50` ~ `10:19`：用户端切换到 `10:30截单`。
4. `10:20` ~ `10:49`：用户端切换到 `11:00截单`。
5. `10:50` 后：用户端切换到 `即时出单`。

#### 7.1.5 统计口径

1. 本页面统计的是用户下单后形成的购物车订单。
2. 用户在同一商家中，购物车提交并支付成功一次，记为 1 个购物车订单。
3. 每个购物车订单按所属截单时段归入对应餐段统计。
4. 已取消订单不计入。
5. 餐品份数为该购物车订单内全部餐品份数之和。

### 7.2 P0-02 配送员主责配置

1. 支持按展区、楼层、展厅、商家筛选责任人。
2. 支持多选配送员后批量配置展厅点位与责任商家。
3. 配置提交采用全量覆盖逻辑。

### 7.3 P0-03 点位-商家配置

1. 支持按展区、楼层联动筛选点位。
2. 支持为点位绑定多个商家。
3. 支持快捷搜索商家并回显已绑定关系。

### 7.4 P0-04 配送员名册

1. 支持按姓名、手机号、角色、状态筛选账号。
2. 支持新增配送员、停用/恢复账号、配置角色。
3. 必须至少保留一个角色。

## 8. 关键交互要求（用于高保真原型）

1. 页面采用 Tab 分区，避免单页信息过载。
2. 统一三层筛选器：展区 -> 楼层 -> 展厅，选项同级联动。点位商家配置Tab需单独增加提示：“此处展厅点位排期仅能选择此处配置的对应商家。”。
3. **全局防重叠渲染控制**：页面采用全局流式自适应弹性布局。配置弹窗基准项信息（如人员姓名）采用纯文本挂载。长列表候选库（如弹窗商家池）标配实时搜索框输入挂载。
4. `配送员主责配置` 台采用同屏配置查询交互，支持：
   - 表格上方提供原生的自动补全（Autocomplete）选择器快速找商家。
   - 包含【分拣签收员】角色的行自带权限，不可进行点位和商家的勾选配置。
   - 弹窗内置“查询人员配置全景”入口（文字链交互），按“当前负责人数”升序排列全局商家和展厅，确保漏配项（零人）永远最高优露出。
5. 手机号重复时前端即时验证，若是由于历史禁用造成的冲突，支持弹窗唤醒复用。
6. `订单实时监控` 页面采用极简表格表达，不额外堆叠说明卡；截单前后差异仅通过列级轻量视觉区分，不增加重复状态文案。

## 9. 数据指标（首期）

1. 点位商家配置完成率（已配置点位数 / 总点位数）。
2. 配送员主责配置覆盖率（已配置主责配送员数 / 启用配送员数）。
3. 配置错误拦截率（重复手机号、重复映射、非法作用域）。
4. 查询命中时延（P95）。
5. 订单实时监控接口刷新时延（P95）。

## 10. 验收标准

1. 新增配送员时，重复手机号被阻断，且提示明确。
2. 批量配置主责时，能正确生成 `AREA/FLOOR/HALL` 作用域记录。
3. 查询任一展厅时，能返回展厅级 + 楼层级 + 展区级责任人。
4. 点位绑定商家后，下游排期接口仅返回已绑定商家。
5. 所有增删改操作都有审计日志。
6. `订单实时监控` 中，任一时刻仅当前开放餐段为主增长列；未开放餐段无订单；已切走餐段仅展示沉淀量。

## 11. 风险与应对

| 风险 | 影响 | 应对策略 |
|---|---|---|
| 展馆字典变更频繁 | 配置失效或漏配 | 字典接口按版本号拉取，配置表保留软引用与快照 |
| 批量配置误操作 | 大范围错误覆盖 | 提交前二次确认 + 变更预览 + 审计回滚 |
| 手机号复用或错填 | 账号冲突 | 前后端双重唯一校验 + 删除二次确认 |
| 主责继承理解偏差 | 查询结果歧义 | 结果页展示命中层级与规则说明 |
