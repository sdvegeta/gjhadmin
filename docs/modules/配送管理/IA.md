# 广交会项目 - 后台配送管理界面 信息架构（IA）

> 版本：V0.5  
> 日期：2026-04-09  
> 对应文档：`后台配送管理界面_PRD.md`、`后台配送管理界面_Spec.md`  
> 目标：为高保真原型提供页面清单、结构层级、路由与关键信息流

## 1. IA 目标与边界

### 1.1 目标

1. 明确后台配送管理子项目的页面全景与导航关系。
2. 明确每个页面承载的信息对象、操作动作、输入输出。
3. 让高保真原型可直接映射到前端路由与组件拆分。

### 1.2 边界

1. 本 IA 只覆盖“后台配送管理界面”。
2. 不覆盖“后台配送统计页”“配送小程序筛选模块”。
3. 不展开视觉规范（颜色、栅格、品牌样式），只定义信息结构与交互层级。

## 5. 页面清单（Page Inventory）

## P0-01 配送管理首页（容器）

- 页面目标：统一承载功能 Tab。
- 核心信息：
1. 当前激活 Tab
- 核心动作：
1. 切换业务 Tab
- 产出影响：
1. 进入对应业务工作台

## P0-02 订单实时监控（Tab1，独立页面模块）

- 页面目标：在用户餐段截单前，让配送负责人按展区查看各商家的实时订单量与餐品份数。
- 核心信息对象：
1. 顶部简洁信息条（最新刷新、60秒自动刷新、立即刷新）
2. 展区分组列表
3. 展开后的商家实时监控表（按截单时段横向展示）
4. 展区小计行
- 核心动作：
1. 切换演示时间（原型态）
2. 手动刷新
3. 展区折叠/展开
- 关键反馈：
1. 与上次刷新相比变化的数字标红
2. 当前开放餐段列轻量强调
3. 已切走餐段列轻量降权

### 5.2.1 结构拆解

1. 页面头部：仅保留刷新相关信息，不保留额外说明卡。
2. 展区层：A/B/C/D 区分组。
3. 商家层：展开后按商家查看横向时段表。
4. 汇总层：每个展区提供小计行。

### 5.2.2 信息层级

1. 一级信息：当前开放餐段所在列。
2. 二级信息：各商家订单数。
3. 三级信息：餐品份数、变化高亮、小计。

## P0-03 点位-商家配置（Tab3）

- 页面目标：维护点位与商家的多对多映射关系。
- 核心信息对象：
1. 顶层筛选域（展区/楼层，联动）及刚性排期提示文案
2. 全高铺展开流、无内生滚动条点位列表（含“操作”动作栏）
3. 候选商家关联库（自带快捷键入式信息流检索）
4. 当前映射对应展示（已绑定/未绑定）
- 核心动作：
1. 选择点位
2. 批量勾选商家并保存
3. 删除已有映射
- 关键反馈：
1. 去重提示（同一点位重复绑定商家）
2. 保存成功提示与差异预览

## P0-04 配送员名册（Tab4）

- 页面目标：维护配送员账号与角色身份。
- 核心信息对象：
1. 配送员列表（姓名、手机号、多角色标签、账号状态、最后变更线）
2. 筛选组（搜索输入、角色快筛、状态快筛）
3. 【新增成员】表单（手机号与姓名）
4. 【配置角色】集中弹窗：人员姓名正常文本展示；角色候选项采用独立交互卡片格式，并明确附带说明文案（配送员：负责将餐品从商家配送到集散地... / 分拣签收员：负责在集散地签收餐品并按展厅分拣餐品 / 配送主管：负责关注配送）。
- 核心动作：
1. 录入新成员（自带默认角色 COURIER）
2. 停用账号（软删） / 恢复账号（包含重复检测原地激活）
3. 配置角色（至少保留一个角色）
- 关键反馈：
1. 配置角色不合法（防空）时的阻断
2. 停用/恢复后的动态状态徽标变换

## P0-05 配送员主责配置（Tab2）

- 页面目标：极简清爽地按地理层级回溯责任人概况并支持主责分配的闭环界面。
- 核心信息对象：
1. 当前责任状态全高表格（姓名、手机号、状态、角色、主责展区/楼层/展厅、主责商家）
2. 顶部的全领域筛选检索（含区、层、点、商家名模糊搜索）
- 核心动作：
1. 依赖全局顶部下拉查询与响应。
2. 从列表左测打勾选取人员
3. 激活并点击面板浮层：【配置展厅点位】、【配置责任商家】执行统一完全覆盖逻辑。
- 关键反馈：
1. 操作提交前的全景二次确认明细。

## P1-06 审计日志（辅助页/抽屉）

- 页面目标：追踪配置变更，支持追责与回溯。
- 核心信息对象：
1. 操作人、时间、对象、变更前后
- 核心动作：
1. 按对象筛选日志
2. 查看详情快照

## 7. 关键任务流（Task Flow IA）

### 7.1 任务流A：查看订单实时监控

```mermaid
flowchart LR
    A["进入Tab1 订单实时监控"] --> B["查看顶部最新刷新与倒计时"]
    B --> C["按展区展开商家明细"]
    C --> D["观察当前开放餐段列的订单量/餐品份数"]
    D --> E["发现变化数字标红，判断是否需要提前布人"]
```

### 7.2 任务流B：点位绑定商家

```mermaid
flowchart LR
    A["进入Tab1"] --> B["选择展区/楼层/展厅"]
    B --> C["加载点位与商家候选"]
    C --> D["勾选商家并保存"]
    D --> E{"是否重复映射?"}
    E -->|是| F["提示冲突并阻断"]
```

### 7.3 任务流C：新增配送员并配置主责

```mermaid
flowchart LR
    A["进入Tab4录入新成员"] --> B{"手机号唯一?"}
```

```mermaid
flowchart LR
    A["进入Tab 配送员主责配置"] --> B["选择条件筛选项"]
    B --> C["执行查询"]

    D --> E{"有结果?"}
```

## 8. 页面间数据与上下文传递

1. 顶部筛选条件为“全局上下文”，Tab 切换不清空。
2. 责任配置台保存后，当前责任状态区应可立即命中新配置（最终一致性不超过数秒）。
3. Tab1 配置变更后，需提示“影响下游排期商家可见范围”。
4. 订单实时监控页不继承其他配置 Tab 的筛选器，以展区分组表为主。

补充：
5. `订单实时监控` 作为独立页面模块，不依赖配送管理首页顶部通栏统计卡。

## 9. 状态与反馈架构（IA 级）

每个页面至少覆盖：
1. Loading（首次加载/提交中）
2. Empty（无数据）
3. Error（请求失败）
4. PermissionDenied（无权限）
5. Confirm（删除/覆盖提交二次确认）

## 10. 权限视角信息架构

| 角色 | 可见页面 | 可执行动作 |
|---|---|---|
| 超级管理员 | 全部 | 全部增删改查 + 日志 |
| 运营配置人员 | 各业务 Tab + 日志 | 点位映射、人员角色维护、责任范围覆盖 |
| 区长/主管 | 查询页 + 日志 | 人力查询、巡检 |
