# 配送管理 - 订单实时监控 PRD

> 版本：V0.1  
> 日期：2026-04-09  
> 所属模块：运营后台 / 配送管理 / 订单实时监控

## 1. 页面目标

让配送团队负责人在用户餐段截单前，按展区查看各商家的实时订单量与餐品份数，用于提前安排运力。

## 2. 核心口径

1. 本页面统计的是用户下单后形成的购物车订单。
2. 用户在同一商家中，购物车提交并支付成功一次，记为 1 个购物车订单。
3. 每个购物车订单按所属截单时段归入对应餐段统计。
4. 已取消订单不计入。
5. 餐品份数为该购物车订单内全部餐品份数之和。

## 3. 餐段开放规则

用户端同一时刻只开放一个可下单餐段，因此监控页同一时刻只存在一个主增长列：

1. `08:50` 前：`9点截单`
2. `08:50` ~ `09:49`：`10点截单`
3. `09:50` ~ `10:19`：`10:30截单`
4. `10:20` ~ `10:49`：`11:00截单`
5. `10:50` 后：`即时出单`

补充规则：
1. 已切走餐段仅保留沉淀量。
2. 未开放餐段默认无订单。

## 4. 页面结构

1. 顶部信息条：
   - 最新刷新
   - 自动刷新倒计时（60秒）
   - 立即刷新
2. 展区分组列表：
   - A区 / B区 / C区 / D区
3. 展开后的商家表格：
   - 商家
   - `9点截单`
   - `10点截单`
   - `10:30截单`
   - `11:00截单`
   - `即时出单`
4. 展区小计行

## 5. 视觉与交互

1. 页面采用极简表格表达，不增加解释型说明卡。
2. 当前开放餐段列轻量强调。
3. 已切走餐段列轻量降权。
4. 与上次刷新相比发生变化的数字标红。
5. 支持按展区折叠/展开查看商家明细。

## 6. 验收重点

1. 任一时刻仅当前开放餐段为主增长列。
2. 未开放餐段展示为 0。
3. 已切走餐段不再继续增长。
4. 60 秒自动刷新与手动刷新均可用。
